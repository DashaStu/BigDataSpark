# BigDataSpark

Анализ больших данных - лабораторная работа №2 - ETL реализованный с помощью Spark

Одним из самых популярных фреймворков для работы с Big Data является Apache Spark. Apache Spark - мощный фреймворк, который предлагает широкий набор функциональности для простого написания ETL-пайплайнов.

Что необходимо сделать? 

Необходимо реализовать ETL-пайплайн с помощью Spark, который трансформирует данные из источника (файлы mock_data.csv с номерами) в модель данных звезда в PostgreSQL, а затем на основе модели данных звезда создать ряд отчетов по данным в одной из NoSQL базах данных обязательно и в нескольких других опционально (будет бонусом). Каждый отчет представляет собой отдельную таблицу в NoSQL БД.

### Аналитические витрины (ClickHouse)
Согласно ТЗ сформированы 6 отчетов:
* `rep1_products`: Выручка, категории, рейтинги и отзывы по продуктам.
* `rep2_customers`: Топ клиентов по тратам и распределение по странам.
* `rep3_time`: Тренды выручки и средний размер заказа по месяцам/годам.
* `rep4_stores`: Топ-5 магазинов и эффективность по городам/странам.
* `rep5_suppliers`: Эффективность поставщиков и средняя цена товаров.
* `rep6_quality`: Анализ корреляции рейтинга, отзывов и объема продаж.

# Инструкция по запуску
```bash
 docker-compose up -d --build
```
А затем можно отследить ход выполнения через логи
```bash
docker logs -f spark_etl_job
```

Чтобы проверить результы выполнения подключаемся к Postgres и ClickHouse

     | POSTGRES_PASSWORD | password |

## Технологический стек
* **Apache Spark 3.5** (обработка данных)
* **PostgreSQL 18** (хранение модели "Звезда")
* **ClickHouse** (хранение аналитических витрин)
* **Docker & Docker Compose** (инфраструктура)
* **Python 3.14** (логика ETL)
